위 alert에서 확인을 누르면, DISCONNECT rest api를 보냅니다.
우선 정상 접속 상황이 아니므로 api 호출 하면 안됩니다.
그리고, userId와 deviceId 값을 잘 못 보내고 있습니다.

RCS Manager(웹화면)에서 전달한 아규먼츠는 아래와 같습니다. 
[44/459/M8/PHOTO/BAY3/M8PPP210/192.168.0.19/admin/admin/true/640/480/100K]
userId: 44, deviceId: 459

해당 상황에서 [X]를 눌러 윈도우를 닫으면, 
DISCONNECT rest api를 보냅니다. 
이 또한 정상 접속 상태 이후, 접속 종료가 아니므로 api 호출 하면 안됩니다.
그리고, userId와 deviceId 값을 잘 못 보내고 있습니다.

정리하자면, 정상 상황에서만, 접속시 CONNECT api 호출을 합니다.
정상 접속이 아닌 예외 상황에서는 alert(error 메시지) 표시 바랍니다.
이때 만약, window 종료시 api 호출 하지 마세요.

정상 접속이 된 상황 이후, 종료시에만 DISCONNECT api 호출을 합니다.
종료시에도 바로 종료 하지 말고, yes or no 질의 후 yes 시 종료로 변경 바랍니다.
